# Emergency SOS — план ручного полевого тестирования

**Версия**: 2026-06-04  
**Реализация**: без переназначения Primary-канала; fixed position через firmware; Waypoint на LongFast-слот

---

## Стенд

| Роль | Устройство |
|---|---|
| SOS-нода | Meshtastic-нода, подключённая к телефону A (MeshTactics) |
| Наблюдатель | Телефон B с Meshtastic или MeshTactics, LongFast включён |
| Оба | В зоне прямой LoRa-видимости или через ретранслятор |

---

## Подготовка

- [ ] GPS на телефоне A заблокирован, ожидаем lock (видим позицию в HUD)
- [ ] Наблюдатель B видит SOS-ноду в списке нод
- [ ] Проверить: LongFast присутствует в конфиге ноды (слот 0 или другой — неважно, фиксируем номер)
- [ ] Записать текущий номер слота LongFast — потребуется при верификации канала Waypoint-ов

---

## TC-01: Базовая активация — нет перезагрузки

- [ ] Нажать SOS → подтвердить
- [ ] Нода **не перезагружается** (BLE-связь не прерывается, HUD не моргает)
- [ ] B получает TextMessage: `"{callsign} просит помощи, координаты: {lat}, {lon}"` на LongFast
- [ ] Через ≤30 сек B видит новую геоточку (Waypoint) на карте с позицией A
- [ ] Канальный слот геоточки = слот LongFast (проверить в Meshtastic app B: Details пакета)

## TC-02: Слот LongFast не совпадает со слотом 0

- [ ] Убедиться, что Waypoint пришёл на правильный слот, а не на Primary (слот 0), если они различаются
- [ ] Distress-сообщение — тоже на LongFast, не на Primary чужого контура

---

## TC-03: Обновление позиции при движении

- [ ] Пройти ≥50м от стартовой точки
- [ ] Ждать ≤30 сек
- [ ] B видит обновление Waypoint-а — позиция сдвинулась
- [ ] Повторить 2–3 раза: каждые 30 сек приходит свежая точка

## TC-04: Fixed position обновляется (опционально)

- [ ] Если есть доступ к firmware debug / Meshtastic CLI: `localPosition` обновляется вместе с Waypoint-ами

---

## TC-05: BLE-дисконнект во время SOS

- [ ] SOS активен, последняя позиция получена наблюдателем
- [ ] Выключить BT на телефоне A (или убить приложение через task manager)
- [ ] Ждать 15–20 мин
- [ ] B получает **POSITION_APP** пакеты от SOS-ноды (~каждые 15 мин, firmware-таймер)
- [ ] Waypoint-поток прекратился (app мёртв) — **ожидаемо**
- [ ] Ключевое: нода автономно транслирует последнюю зафиксированную позицию

## TC-06: Восстановление приложения после дисконнекта

- [ ] Включить BT, перезапустить MeshTactics
- [ ] SOS-флаг восстановился (кнопка в активном состоянии)
- [ ] Waypoint-поток возобновился в течение 30 сек
- [ ] `setFixedPosition` обновляется с актуальными координатами

---

## TC-07: Корректная отмена SOS

- [ ] SOS активен, наблюдатель B регулярно получает Waypoint-ы
- [ ] Нажать отмена SOS → подтвердить
- [ ] B получает TextMessage: `"Пользователь {callsign} отметил, что с ним всё в порядке..."`
- [ ] Новые Waypoint-ы **прекратились** немедленно

## TC-08: Fixed position снят — КРИТИЧНО

- [ ] После отмены ждать 20–25 мин (дольше firmware-таймера ~15 мин)
- [ ] B **не получает** новых POSITION_APP пакетов от SOS-ноды
- [ ] Если получает → `removeOwnFixedPosition` не сработал: **дефект**

## TC-09: Нода не перезагружается при отмене

- [ ] BLE-связь не прерывается во время отмены SOS
- [ ] HUD остаётся в connected-состоянии без разрыва

---

## TC-10: Provide Location заблокирован во время SOS

- [ ] Включить "Provide Location" в настройках ноды до активации SOS
- [ ] Активировать SOS
- [ ] Нода **не** получает `sendPosition` пакеты с телефона (проверить через Meshtastic packet log)
- [ ] Fixed position SOS не обнулён (позиция корректная на B)

## TC-11: Provide Location возобновляется после отмены

- [ ] Отменить SOS
- [ ] "Provide Location" возобновился: нода снова получает GPS с телефона
- [ ] Позиция на B продолжает обновляться

---

## TC-12: SOS без GPS-лока

- [ ] Отключить GPS / зайти в помещение
- [ ] Активировать SOS
- [ ] Distress-сообщение: `"{callsign} просит помощи"` (без координат)
- [ ] Broadcast-цикл запущен, итерации пропускаются (null GPS)
- [ ] Выйти на улицу → GPS lock → в течение 30 сек первый Waypoint и fixed position

---

## Сводный чек-лист

| Критерий | Pass | Fail |
|---|---|---|
| Нет перезагрузки при активации | BLE stable | HUD disconnect |
| Distress на LongFast | Message на правильном слоте | Message на другом канале |
| Waypoint каждые 30 сек | Регулярные точки на B | Нет точек / другой канал |
| После отключения телефона: автономный broadcast | POSITION_APP от ноды на B | Тишина через 20 мин |
| После отмены: нет POSITION_APP через 20+ мин | Тишина | Нода продолжает вещать |
| Нет перезагрузки при отмене | BLE stable | HUD disconnect |
