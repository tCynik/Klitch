# Передача гео-объектов через Meshtastic

**Статус**: Draft — промежуточные результаты обсуждения, реализация не начата

---

## Принципиальная позиция: отказ от совместимости

Данная реализация **намеренно отказывается от обратной совместимости** со сторонними Meshtastic-клиентами. Поля Waypoint используются не по семантике Meshtastic, а переопределяются под нужды MeshTactics. Сторонний клиент получит корректный Waypoint с первой точкой, нечитаемым description и некорректным icon — это приемлемо. Экосистема MeshTactics является замкнутой.

---

## Транспорт

Все гео-объекты передаются как **Meshtastic Waypoint**. Каждый объект помещается в **один пакет** — многопакетная передача не предусмотрена.

### Маппинг полей Waypoint

| Поле Waypoint | Семантика MeshTactics |
|---|---|
| `lat / lon` | Абсолютные координаты якоря (первая точка / центр примитива) |
| `name` | Название объекта (задаёт пользователь) |
| `expire` | Срок жизни объекта |
| `locked_to` | ID ноды-автора (node ID отправителя) |
| `icon` | Тип объекта + цвет + вариант иконки (см. ниже) |
| `description` | `"MT1:<base64(payload)>"` — геометрия и специфичные параметры |

## Поле icon: кодирование типа и атрибутов

`icon: uint32` используется как упакованный дескриптор объекта, а не как emoji codepoint.

```
 31      24  23     16  15      8  7       0
┌──────────┬──────────┬──────────┬──────────┐
│ namespace│   type   │  color   │ variant  │
│  0x4D   │ 00..04   │ 00..0F   │ 00..FF   │
└──────────┴──────────┴──────────┴──────────┘

namespace = 0x4D  — идентификатор MeshTactics ('M')
type      = тип объекта (см. таблицу ниже)
color     = индекс цвета в палитре MeshTactics (4 бита = 16 цветов)
variant   = вариант иконки внутри типа (256 вариантов на тип)
```

### Типы объектов

| type | Название | Описание |
|---|---|---|
| 0 | Point | Точка с иконкой и комментарием |
| 1 | Track | Маршрут (последовательность точек) |
| 2 | Polygon | Полигон (замкнутый контур) |
| 3 | Primitive | Геометрическая фигура |
| 4 | Package | Контейнер точек и примитивов (теоретически) |

## Иконки

Разрабатывается собственный пак иконок для экосистемы MeshTactics. Принципы:
- Ограниченный набор — интерфейс остаётся простым
- Адресация через поле `variant` в `icon`
- Палитра цветов фиксирована (16 цветов), индекс в поле `color`

## Поле description: payload по типу

Общий формат: `"MT1:<base64(payload)>"`.  
Префикс `MT1:` идентифицирует пакет MeshTactics версии 1.

### Point
```
comment: string (остаток байт)
```

### Track / Polygon
```
count:  u8       // число дополнительных точек (не считая якоря)
ends:   u8       // тип законцовки (Track) / окантовка (Polygon)
points: []
  x: int16       // метры на восток от якоря
  y: int16       // метры на север от якоря
comment: string  // остаток байт
```

### Primitive
```
prim_type: u8    // CIRCLE=0, ELLIPSE=1, RECT=2, ARROW=3
params: []       // зависит от prim_type (радиус / стороны / азимут)
comment: string
```

## Локальная система координат

- Якорь (`lat/lon` пакета) = начало координат (0, 0)
- X = восток, Y = север, единица = 1 метр
- Диапазон int16: ±32 767 м ≈ ±33 км от якоря
- Точность: 1 метр
- Приближение: плоская земля, точно до ~50 км

### Конвертация

```kotlin
fun toLocal(anchor: LatLon, point: LatLon): Pair<Int, Int> {
    val metersPerDegLat = 111_320.0
    val metersPerDegLon = 111_320.0 * cos(Math.toRadians(anchor.lat))
    val x = ((point.lon - anchor.lon) * metersPerDegLon).roundToInt()
    val y = ((point.lat - anchor.lat) * metersPerDegLat).roundToInt()
    return x to y
}

fun fromLocal(anchor: LatLon, x: Int, y: Int): LatLon {
    val metersPerDegLat = 111_320.0
    val metersPerDegLon = 111_320.0 * cos(Math.toRadians(anchor.lat))
    return LatLon(
        lat = anchor.lat + y / metersPerDegLat,
        lon = anchor.lon + x / metersPerDegLon
    )
}
```

## Ёмкость и валидация

| Параметр | Значение |
|---|---|
| Макс. payload LoRa-пакета | ~237 байт |
| Overhead Waypoint (id, lat, lon, expire, icon, locked_to, name, headers) | ~40 байт |
| Остаток на description | ~195 символов |
| После Base64 overhead (×3/4) | ~145 байт raw |
| Байт на точку для Track/Polygon (int16 × 2) | 4 байта |
| **Макс. дополнительных точек** | **~27** |
| **Итого точек включая якорь** | **~28** |

### Валидация на стороне отправителя

- Константа `MAX_PAYLOAD_BYTES = 145` проверяется до сериализации
- Для Track/Polygon: константа `MAX_POINTS = 27`, проверяется при создании объекта
- При превышении — ошибка пользователю, отправка не производится
