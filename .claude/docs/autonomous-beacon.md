# Автономный маяк (Autonomous Beacon)

**Date**: 2026-05-12
**Updated**: 2026-05-28
**Status**: Concept — UX-флоу набросан, реализация не начата

---

## Суть фичи

Настройка ноды с встроенным GPS-приёмником на автономную рассылку геопозиции в заданный
приватный канал (контур) — без участия приложения, без активного соединения телефона с нодой.

**Сценарий**: нода закреплена на объекте/транспорте, работает автономно, периодически шлёт
позицию в канал группы. Члены группы видят маяк на карте.

---

## Бизнес-контекст

Бизнес-требование: геолокация должна работать как через телефонный GPS, так и через
встроенный GPS-приёмник ноды.

**Ограничение прошивки Meshtastic**: авто-броадкаст позиции ноды (`position_broadcast_secs`)
всегда идёт через PRIMARY канал (slot 0). Нет штатного механизма направить авто-броадкаст
на secondary канал.

**Следствие**: в текущей архитектуре встроенный GPS ноды не может автономно рассылать
позицию в приватный групповой контур (secondary канал). Только телефонный GPS управляется
приложением и может адресоваться в любой канал.

---

## Предлагаемое решение

Перенастроить ноду-маяк: назначить приватный групповой канал на **slot 0** (PRIMARY).
Тогда прошивка будет автоматически броадкастить позицию в этот канал.

**Компромисс**: такая нода теряет Emergency-канал (LongFast, AQ==) как primary.
Emergency-канал может остаться как secondary (slot 1), но автоматической рассылки позиции
в него уже не будет — только ручная отправка через приложение.

---

## Известные ограничения

| Ограничение | Последствие |
|---|---|
| Нода-маяк без приложения не может слать в Emergency | SOS-геолокация для маяка недоступна в автономном режиме |
| Смена primary = выход из дефолтного LongFast-меша | Нода-маяк не видна обычным Meshtastic-пользователям |
| Требует явной перенастройки slot 0 через приложение | Нельзя сделать маяком ноду без подключения к MeshTactics |

---

## Что настраивается на маяке

Три группы настроек, все через `AdminMessage`:

| Настройка | AdminMessage | Ключевые поля |
|---|---|---|
| Позывной | `set_owner = User(...)` | `long_name`, `short_name` |
| Геолокация | `set_config = Config(position = PositionConfig(...))` | `gps_mode`, `position_broadcast_secs`, `gps_update_interval`, `position_flags` |
| Каналы | `set_channel = Channel(index=0, ...)` | slot 0 → PSK группового контура |

**Порядок**: owner → position config → channels. Смена каналов вызывает перезагрузку ноды.

---

## Подход к соединению

### Remote Admin через меш — ОТКЛОНЁН

Формально: `AdminMessage` с `to = "!beaconNodeId"` идёт через Node A в меш к маяку.
Практически: node-to-node доставка не работает — сообщения не доставляются.
Проблема воспроизводится и в оригинальном Meshtastic-приложении.
Дополнительно: каскад перезагрузок и зависимость от порядка каналов (политика меняется).

### Последовательный BLE-переход — ПРИНЯТ

Сценарий редкий (admin-функция), поэтому переключение BLE допустимо.
UX сглаживается: сканирование начинается до разрыва соединения с основной нодой.

## UX-флоу (набросок)

```
[Экран Сеть] → кнопка "Настроить автономный маяк"
                ↓
[Beacon Setup Screen]
  — соединение с Node A ДЕРЖИМ
  — BluetoothLeScanner.startScan() ЗАПУСКАЕМ сразу
  — список Meshtastic-нод заполняется в реальном времени
                ↓
  пользователь выбирает ноду из списка
                ↓
  disconnect Node A → connect Beacon (разрыв только здесь)
                ↓
  [флоу настройки — отдельный таск]
                ↓
  кнопка "Готово" / назад
                ↓
[Экран Сеть] + auto-connect к Node A (lastConnectedDeviceAddress из AppSettings)
```

**Ключевое**: `BluetoothLeScanner` и `BluetoothGatt` — независимые подсистемы Android BLE.
Сканирование работает поверх активного GATT-соединения без конфликтов.

**Отмена без выбора**: сканирование стоп, соединение с Node A не трогалось — reconnect не нужен.

## Вопросы для проработки

- UX: как пользователь выбирает ноду-маяк и привязывает контур?
- Можно ли вернуть ноду из режима маяка обратно?
- Нужен ли отдельный тип контура «маяк» или это режим существующего контура?
- Как отобразить ноду-маяк на карте отдельно от живых участников?
- Проверить: поддерживает ли прошивка `position_precision > 0` на нескольких каналах
  одновременно — если да, то авто-броадкаст может идти на несколько каналов и компромисс
  по Emergency частично снимается.
- Детали флоу настройки (шаг между выбором ноды и "Готово") — отдельный таск.
